Шпоргалка по GIT

  • git config --global user.name blabla - добавление имени пользователя после установки GIT. Это нужно для понимания кто подписывает комит
  • git config --global user.email blabal@bla - добавление имэйла пользователя после установки GIT. Это нужно для понимания кто подписывает комит
  • git init - создать пустой GIT
  • git status - просмотреть какие файлы которые имеют различия с файлом индекса и текущим состоянием
  • git add - подготовить файты к комиту
  • git commit - создать комит описывающий новые изменения
  • git log - отображает изменения каждого комита
  • git checkout - обновляет файлы в рабочем дереве в соответствии с версией в индексе или указаном дереве
  • git branch - показывает все ветки и текущую подсвечивает зеленым
  • git push - отправить данные из репозитория
  • git pull - получить данные в репозиторий
  • git merge - Эта команда используется для слияния веток
  • git clone - клонирует репозиторий в новый каталог

Advanced

  • git revert - отменяет новый комит если он был ненужен
  • git reset - отменяет все изменения в рабочем каталоге
  • git rebase - работает как и merge но не сохранияет историю (последовательность) комитов. Лучше не использовать в командной работе, а для локальных веток
  • git cherry-pick - перетянуть комит из другой ветки в свою
  • git stash - если нам не нужно делать комит, но нужно сохранить текущюю версию изменений.
